# System scope is opportunity-discovery, not autonomous building

The system could either (A) surface ranked pain points with a solution sketch for a human to decide on, or (B) autonomously generate a working prototype for the top pain point(s). We chose A: judging whether a pain point is worth building for is a subjective, market-feel call that's hard to automate well, and validating with a written brief is far cheaper than validating with a built prototype. B may be a later stage that consumes A's output, but is out of scope for the core loop.
